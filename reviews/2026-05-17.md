## Critical Issues (Production-blocking)

1. **No rate limiting on `/api/auth/login`** — brute-force in seconds on LAN
2. **Legacy token fallback grants admin silently** — any JWT that fails `userId` check → admin. Permanent backdoor.
3. **JSONB race condition on issues** — `investigation_steps` and `notes` use read-modify-write on full JSONB array. Concurrent requests lose data silently.
4. **Schema drift** — `schema.sql` doesn't include `users`, `project_members`, `audit_events`, or embedding columns on commands/issues. Fresh setup ≠ migrated setup.
5. **No Ollama request timeouts** — hung Ollama process blocks Express handler forever; exhausts connection pool under load.
6. **SSRF on URL import** — `POST /api/documents/url` fetches arbitrary URLs; no whitelist prevents `http://localhost`, internal IPs.
7. **Credentials hardcoded in docker-compose.yml** — `POSTGRES_PASSWORD: devbrain` in plaintext.
8. **Embedding failures silently swallowed** — `.catch(() => {})` on all embed calls; failed embeddings leave records invisible to semantic search with no repair path.

## Security Issues

| Issue | Severity | File | Detail |
|-------|----------|------|--------|
| No rate limiting on login | Critical | `routes/auth.ts` | Unlimited attempts, no backoff |
| Legacy token → admin fallback | Critical | `middleware/auth.ts:47` | Any JWT with missing `userId` gets `role: 'admin'` |
| JWT tokens in localStorage | High | `lib/api.ts:7,13` | Vulnerable to XSS; HttpOnly cookies are correct |
| No JWT `iss`/`aud` claims | High | `routes/auth.ts:12-14` | Tokens from other services accepted |
| SSRF on URL document import | High | `routes/documents.ts:166` | No domain whitelist; server fetches attacker-controlled URLs |
| Dynamic SQL column names | High | `routes/commands.ts:151`, `routes/documents.ts:214` | `Object.keys(updates)` → column names in SQL; no explicit whitelist |
| Timing attack on login | Medium | `routes/auth.ts:38,74` | Different code paths for "user not found" vs "wrong password" leak username existence |
| Admin password reset, no notification | Medium | `routes/users.ts:82-92` | Admin resets any password silently; no audit log entry |
| Password changes not audit-logged | Medium | `routes/auth.ts` | `POST /auth/change-password` has no `logAudit()` call |
| Docker secrets in plaintext | Medium | `docker-compose.yml` | `POSTGRES_PASSWORD`, `JWT_SECRET` should come from env or secrets |
| No HTTPS enforcement | Medium | `server/index.ts` | No redirect or HSTS header; credentials sent in plaintext over HTTP |

## Architecture Issues

| Issue | Severity | Detail |
|-------|----------|--------|
| Custom event routing (`devbrain:navigate`) | High | No browser history, no deep links, no bookmarkable URLs; back/forward broken |
| JSONB mutable arrays (steps, notes) | High | Read-modify-write pattern loses data under concurrent writes; should be normalized tables |
| Schema drift between `schema.sql` and migrations | High | Fresh install ≠ migrated install; no single source of truth |
| Fire-and-forget embeddings | High | `.catch(() => {})` on all embed calls; no retry queue, no visibility, no repair |
| No Ollama timeouts | High | `AbortController` needed; hung request blocks indefinitely |
| No error boundaries in React | Medium | Any component crash = blank white app |
| No API versioning | Medium | `/api/` namespace; breaking changes have no migration path |
| State not URL-driven | Medium | Route, selected project, open panel all ephemeral; refresh resets everything |
| Partial Zustand persistence | Low | Only `selectedId` persisted; projects list always refetched on reload |
